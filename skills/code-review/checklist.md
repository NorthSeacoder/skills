# Code Review Checklist

## 1. 范围与目标

- 改动是否对准当前目标
- 是否混入无关重构
- 是否引入未说明的新依赖

## 2. 功能正确性

- 主路径是否成立
- 条件分支是否正确
- 返回值与错误路径是否一致

## 3. 回归风险

- 共享模块是否被改动
- 老调用方是否会受影响
- 配置默认值是否改变

## 4. 边界与异常

- 空值、空列表、未命中场景
- 超时、失败、重试
- 并发、重复提交、幂等性

## 5. 数据与状态

- 状态更新是否完整
- 读写顺序是否合理
- 数据模型改动是否同步到调用方

## 6. 测试

- 是否新增或更新必要测试
- 测试是否覆盖核心路径
- 回归 bug 是否有对应测试

## 7. 配置与工程化

- workflow / CI 改动是否合理
- 构建、lint、typecheck 是否受影响
- 环境变量、权限、缓存策略是否安全

## 8. 发布影响

- 是否需要迁移或版本说明
- 是否需要更新 changelog / docs
- 是否适合直接合并或发布
