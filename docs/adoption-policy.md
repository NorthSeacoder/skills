# 纳入策略

## 纳入条件

只有同时满足下面条件的 skill，才应该进入这个仓库：

- 能解决重复出现的问题
- 对我自己的工作流确实有价值
- 我愿意长期维护
- 不依赖不可见的隐式基础设施也能理解和使用

## 优先纳入

优先考虑以下类型：

- 轻量
- 可组合
- 对环境依赖弱
- 触发条件和输出边界清楚

## 不建议纳入

以下类型原则上不进入本仓库：

- 强依赖公司内网或专有系统
- 没有外部运行时栈就无法成立
- 已被全局稳定安装的第三方 skill 完整覆盖
- 过于宽泛，超出个人仓库可维护边界

## adopted skill 规则

外部 skill 一旦被拷入本仓库，就视为本地维护资产。

这意味着：

- 命名要避免冲突
- 本地改动要有记录
- 上游只作为参考，不再视为当前真实来源
